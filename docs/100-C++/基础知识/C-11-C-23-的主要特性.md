## 1. C++14（2014）：小幅增强与改进**
C++14 并没有像 C++11 那样带来颠覆性变化，而是作为一个 **补充标准**，主要在语法和标准库上做了“平滑优化”。

---

### 1.1. 语言增强**
#### 1.1.1. 1. 泛型 Lambda（Generic Lambdas）**
C++11 的 Lambda 捕获支持，但参数需要显式指定类型；C++14 引入 auto，让 Lambda 参数能自动推导。

```cpp
#include <iostream>`
#include `<vector>`
#include `<algorithm>`

int main() {
    std::vector<int> v{1,2,3};

    // 泛型 lambda：参数类型自动推导
    auto print = [](auto x) { std::cout `<< x << " "; };

    std::for_each(v.begin(), v.end(), print);
}
```

---

#### 1.1.2. 2. 函数返回类型推导**
C++11 需要 ->` decltype(...) 指定返回类型，C++14 可以直接省略。

```cpp
#include `<iostream>`

// C++14 起，编译器可自动推导返回类型
auto add(int a, int b) {
    return a + b; // 推导为 int
}

int main() {
    std::cout `<< add(3, 4) << std::endl;
}
```

---

#### 1.1.3. 3. 变量模板（Variable Templates）**
允许模板参数化变量，而不仅仅是函数或类。

```cpp
#include <iostream>`

`template`<typename T>`
constexpr T pi = T(3.1415926535897932385L);

int main() {
    std::cout `<< `pi<double>` `<< "\n"; // double 类型的 pi
    std::cout << `pi<float>`  `<< "\n"; // float 类型的 pi
}
```

---

#### 1.1.4. 4. Relaxed**  **constexpr**
C++11 的 constexpr 限制太严格（只能有一行 return）；C++14 放宽了，可以有多行逻辑。

```cpp
#include <iostream>`

constexpr int factorial(int n) {
    int result = 1;
    for (int i = 1; i <= n; ++i) result *= i;
    return result;
}

int main() {
    constexpr int f = factorial(5); // 编译期计算
    std::cout `<< f << std::endl;    // 输出 120
}
```

## 3. C++20（2020）：概念、模块与协程的时代**
C++20 带来了 **语言核心特性的大跃进**（Concepts、Modules、Coroutines），同时在标准库中引入 **Ranges、格式化、日历库** 等强力工具。

### 3.1. 语言增强**
#### 3.1.1. 1. Concepts（概念）**
用来约束模板参数，取代 enable_if，让泛型更可读。

```cpp

    return a + b;
}

    // std::cout << add(3.14, 2.0); // ❌ 编译错误
```

---

#### 3.1.2. 2. Ranges（区间）**
更优雅地处理容器和算法，支持 **管道操作符 |**。


#### 3.1.3. 3. 协程（Coroutines）**
轻量级协作式任务，可以写异步生成器。

```cpp

        auto initial_suspend() { return std::suspend_always{}; }

        auto yield_value(int value) {
            current_value = value;
            return std::suspend_always{};
        }
        void return_void() {}
        void unhandled_exception() { std::exit(1); }

    promise_type* p;
    Generator(promise_type* p): p(p) {}
    bool move_next() { return p->current_value++, false; }

    // 简化版生成器例子太长，这里略写
    std::cout `<< "C++20 协程支持 async/await/generator\n";
```

>` ⚡ 实际中常用 co_yield、co_return、co_await，配合 std::generator 或三方库。

## 4. C++23（2023）：进一步完善现代 C++**
C++23 没有引入像 C++20 的 **Concepts/Modules/Coroutines** 那样的重量级新特性，而是专注于 **补齐短板、提升易用性**，让语言更接近现代开发需求。

---

### 4.1. 语言增强**
#### 4.1.1. 1.**  **deducing this**
允许成员函数把 this 当作参数使用，支持更好的链式调用。

```cpp
#include `<iostream>`

struct Foo {
    int value{0};

    auto operator()(this Foo self, int x) {
        self.value += x;
        return self; // 返回修改后的副本
    }
};

int main() {
    Foo f;
    auto g = f(10)(20); 
    std::cout `<< g.value << std::endl; // 输出 30
}
```

---

#### 4.1.2. 2. 模块化增强**
C++23 增强了 **模块**，允许导入标准库模块，如：

```cpp
import std;  // 导入整个标准库模块
```

（不同编译器支持情况可能不一，GCC/MSVC/Clang 正在逐步实现）

#### 4.1.3. 3. 扩展的** **constexpr**
